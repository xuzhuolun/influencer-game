# SDD 实现技能

按照规格和待办列表精确构建已规划的内容。

## 何时使用

- 执行已规划的实现
- 从规格生成代码
- 按方案构建功能
- 系统化遵循待办列表

## 协议

### 步骤 1：加载方案
按顺序阅读：
1. `specs/active/[task-id]/plan.md` - 技术方案
2. `spec.md` - 需求和验收标准
3. `tasks.md` - 任务分解
4. `todo-list.md` - 执行清单

### 步骤 2：系统化执行

1. **开始任何工作前阅读整个列表**
2. **按顺序执行** - 尊重依赖链
3. **标记完成** - 立即将 `- [ ]` 改为 `- [x]`
4. **记录阻塞** - 不要静默跳过，使用 `[BLOCKED: 原因]`

### 步骤 3：遵循模式
参考 `references/patterns.md` 了解：
- 项目特定规范
- 常见实现模式
- 代码质量标准

### 步骤 4：跟踪进度
使用 `scripts/progress.sh` 可视化完成状态。

### 步骤 5：报告完成

```markdown
## 实现摘要

### 已完成
- [x] 任务 1: 描述
- [x] 任务 2: 描述

### 创建/修改的文件
- `path/to/file.ts`: [用途]

### 遇到的阻塞
- [阻塞和解决方案]

### 发现
- [应该更新规格的内容]

### 下一步
- [建议的后续行动]
```

## 代码质量标准

- 遵循现有项目模式（先检查类似文件）
- 匹配代码库中使用的命名规范
- 适当处理错误
- 考虑规格中的边界情况
- 编写可测试的代码

## 要避免的反模式

- 不解释就跳过任务
- 没有完成就标记为已完成
- 与计划不同的实现却不说明原因
- 完成工作后不更新复选框
- 忽略阻塞而不是记录它们

## 集成

- 完成后，`sdd-verifier` 子代理验证工作
- 发现触发 `sdd-evolve` 技能进行规格更新
- 对不明确的需求使用提问工具
