# 规则升级 - 将反例转化为正式规则

> **触发时机**: 用户确认要升级规则时手动触发
> **前置条件**: 建议先执行 `rules_review` 获取分析报告

## 📋 执行任务

### 1. 确认升级目标

用户指定要升级的反例或问题类型：

{{selection}}

**如果未指定**：处理 L3-project-badcase.mdc 中所有 `[ ] 待转化` 状态的反例。

### 2. 确定目标层级

根据问题性质判断：
- **L1** - 通用工程规则（适用于所有 UE5 开发）
- **L2** - 子系统规则（特定子系统如 RPC、GAS、调试）

### 3. 创建/更新规则

**如果是新规则**：
- 在对应层级创建新文件（如 `L2-ue5-xxx.mdc`）
- 必须包含 `[Constraint]` 约束条款
- 更新 INDEX.mdc 索引

**如果是补充现有规则**：
- 在现有规则文件中添加新的约束条款

### 4. 更新反例状态

将 L3-project-badcase.mdc 中对应反例的状态改为：
```markdown
> **状态**: `[x] 已转化` → 见 [目标规则文件]
```

---

## ⚡ 开始执行

根据用户指定的反例，执行规则升级。
