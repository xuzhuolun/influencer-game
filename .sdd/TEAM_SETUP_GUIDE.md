**功能:** Cursor 2.1 团队命令集成

**状态:** ✅ 就绪

## 概述

Cursor 2.1 允许你在仪表板中定义自定义命令，这些命令会自动与整个团队共享。本指南展示如何为团队设置 SDD 命令。

## 优势

### 对于团队
- ✅ **一致的工作流程** - 每个人使用相同的 SDD 命令
- ✅ **集中管理** - 一次更新命令，应用于所有人
- ✅ **轻松入职** - 新团队成员自动获得命令
- ✅ **无需本地设置** - 无需在本地管理 `.cursor/commands/` 文件
- ✅ **版本控制** - 命令在仪表板中管理

### 对于管理员
- ✅ **控制** - 定义团队标准
- ✅ **更新** - 更改命令，团队自动获得更新
- ✅ **可见性** - 查看谁在使用哪些命令
- ✅ **合规** - 确保团队遵循 SDD 方法论

## 设置流程

### 步骤 1：访问团队仪表板

1. 前往 [Cursor 仪表板](https://cursor.com/dashboard)
2. 导航到 **团队设置**
3. 选择 **团队命令** 部分

### 步骤 2：添加 SDD 命令

**选项 A：从仓库导入**

如果你的团队在仓库中有 SDD 命令：

1. 点击 **导入命令**
2. 选择仓库
3. 选择 `.cursor/commands/` 目录
4. 命令自动导入

**选项 B：手动创建**

对于每个 SDD 命令：

1. 点击 **新建命令**
2. 输入命令名称（例如 `brief`）
3. 从 `.cursor/commands/brief.md` 粘贴命令内容
4. 保存命令

### 步骤 3：配置命令访问权限

**设置权限：**
- **所有团队成员** - 每个人都可以使用
- **特定角色** - 限制为某些角色
- **只读** - 查看但不能修改

**设置可见性：**
- **公开** - 对所有团队成员可见
- **私有** - 仅管理员可见
- **已归档** - 隐藏但未删除

### 步骤 4：与团队共享

**自动共享：**
- 命令自动出现在团队成员的界面
- 团队成员无需操作
- 设置后立即可用

**通知：**
- 团队成员收到关于新命令的通知
- 命令出现在 `/` 下拉菜单中
- 使用情况在仪表板中跟踪

## 命令配置

### 命令元数据

对于每个命令，配置：

**基本信息：**
- **名称：** 命令标识符（例如 `brief`）
- **描述：** 命令的功能
- **类别：** SDD 主要、SDD 高级、SDD 规划

**内容：**
- **来源：** 从 `.cursor/commands/[command].md` 复制
- **格式：** Markdown
- **变量：** 支持占位符

**设置：**
- **可见性：** 公开/私有/已归档
- **权限：** 谁可以使用/修改
- **版本：** 跟踪更改

### 配置示例

```yaml
命令: brief
名称: 简报功能规划
描述: 创建 30 分钟功能简报用于快速开发
类别: SDD 主要
来源: .cursor/commands/brief.md
可见性: 公开
权限: 所有团队成员
版本: 2.1
```

## 与本地命令集成

### 混合方法

**团队命令：**
- 标准 SDD 工作流程
- 团队范围一致性
- 集中管理

**本地命令：**
- 项目特定命令
- 个人偏好
- 实验性功能

**两者协同工作：**
- 团队命令首先出现
- 本地命令补充
- 无冲突

### 优先级顺序

1. **团队命令** - 标准工作流程
2. **本地命令** - 项目特定
3. **内置命令** - Cursor 默认

## 故障排除

### 命令未出现

**检查：**
- Cursor 版本（需要 2.1+）
- 团队成员身份已确认
- 仪表板权限已设置
- 命令可见性设置

**解决方案：**
- 更新 Cursor
- 验证团队访问权限
- 检查仪表板设置
- 联系管理员

### 命令不工作

**检查：**
- 命令内容有效
- Markdown 格式正确
- 变量正确定义
- 权限正确设置

**解决方案：**
- 审查命令内容
- 检查格式
- 验证权限
- 在仪表板中测试

### 更新未出现

**检查：**
- Cursor 同步状态
- 网络连接
- 仪表板更改已保存
- 团队成员 Cursor 版本

**解决方案：**
- 刷新 Cursor
- 检查连接
- 验证更改已保存
- 如需要则更新 Cursor
