# SDD 审计检查清单

对照规格审计实现的完整检查清单。

## 功能性需求

### 核心功能
- [ ] 所有用户故事已实现
- [ ] 每个故事的验收标准已满足
- [ ] 快乐路径正常工作
- [ ] 错误路径优雅处理
- [ ] 规格中的边界情况已处理

### API 符合度
- [ ] 端点匹配规格契约
- [ ] 请求/响应 schema 正确
- [ ] 错误响应遵循标准格式
- [ ] 认证/授权已执行
- [ ] 限流已实现（如规格指定）

### 数据完整性
- [ ] 数据模型匹配规格
- [ ] 验证已就位
- [ ] 必填字段已强制
- [ ] 关系正确
- [ ] 迁移可逆

## 非功能性需求

### 性能
- [ ] 响应时间在目标范围内
- [ ] 无 N+1 查询
- [ ] 指定位置已实现缓存
- [ ] 大数据集有分页
- [ ] 重任务使用异步操作

### 安全
- [ ] 所有用户输入有输入验证
- [ ] SQL 注入防护
- [ ] XSS 防护
- [ ] CSRF 保护
- [ ] 密钥未硬编码
- [ ] 敏感数据已加密
- [ ] 认证令牌安全

### 可靠性
- [ ] 错误处理全面
- [ ] 日志足够调试
- [ ] 优雅降级
- [ ] 适当位置有重试逻辑
- [ ] 断路器（如规格指定）

## 代码质量

### 结构
- [ ] 遵循项目模式
- [ ] 单一职责原则
- [ ] DRY - 无不必要重复
- [ ] 清晰的关注点分离
- [ ] 适当的抽象级别

### 可读性
- [ ] 有意义的命名（变量、函数、类）
- [ ] 函数少于 50 行
- [ ] 复杂逻辑有注释

### 测试
- [ ] 业务逻辑有单元测试

- [ ] 边界情况已测试
- [ ] 错误路径已测试
- [ ] 测试覆盖率达到目标

## 文档
